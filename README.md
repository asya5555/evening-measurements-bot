# Вечерние измерения

Персональный Telegram-бот-дневник на русском языке. Основной сценарий — заполнение прошедшего календарного дня: по умолчанию бот в 13:00 предлагает зафиксировать вчера, а в 18:00 может один раз ненавязчиво напомнить. MVP собирает наблюдения, извлекает структурированные данные, задаёт только недостающие вопросы, показывает запись на подтверждение, поддерживает голос/фото как входы, готовит аналитику, напоминания и экспорт.

## Стек

- Python 3.12
- FastAPI
- aiogram 3
- PostgreSQL
- SQLAlchemy 2 и Alembic
- Pydantic 2
- OpenAI Python SDK, Responses API и structured outputs
- APScheduler-ready сервис напоминаний
- Docker Compose
- pytest, ruff, mypy

## Быстрый запуск

```bash
cp .env.example .env
docker compose up --build
```

После запуска API доступен на `http://localhost:8000/health`.

## Что нужно заполнить в `.env`

- `TELEGRAM_BOT_TOKEN` — токен бота от BotFather.
- `TELEGRAM_ALLOWED_USER_IDS` — Telegram user ID основного пользователя, например `123456789`.
- `TELEGRAM_ALLOWED_USERNAMES` — временная альтернатива ID, например `asya5555`.
- `TELEGRAM_WEBHOOK_SECRET` — случайная строка для защиты webhook.
- `PUBLIC_WEBHOOK_BASE_URL` — публичный HTTPS URL сервиса.
- `AUTO_SET_WEBHOOK` — `true` для автоматической установки Telegram webhook при старте.
- `OPENAI_API_KEY` — ключ OpenAI API.
- `DATABASE_URL` — PostgreSQL DSN.

Секреты не хранятся в базе и не должны попадать в git.

## Telegram token

1. Откройте Telegram.
2. Напишите `@BotFather`.
3. Выполните `/newbot`.
4. Скопируйте токен в `TELEGRAM_BOT_TOKEN`.

## OpenAI key

Создайте API key в OpenAI dashboard и положите его в `OPENAI_API_KEY`.

## Команды

- `/start` — первичная настройка.
- `/today` — заполнить сегодняшний день.
- `/yesterday` — заполнить вчерашний день.
- `/add` — добавить информацию к существующей записи.
- `/edit` — исправить запись.
- `/history` — последние записи.
- `/week` — недельная аналитика.
- `/month` — месячная аналитика.
- `/export` — выгрузка данных.
- `/settings` — настройки.
- `/help` — помощь.
- `/cancel` — отмена сценария.
- `/delete_my_data` — удаление данных с двойным подтверждением.

## Основной сценарий

Каждый день бот предлагает заполнить предыдущий календарный день:

- первое напоминание по умолчанию в `13:00`;
- одно повторное напоминание по умолчанию в `18:00`;
- если запись уже есть, бот должен предлагать дополнить или изменить её;
- пустые записи за пропущенные дни автоматически не создаются.

Если пользователь пишет «сегодня», «вчера», «позавчера», «ко вчера», «в субботу» или похожую фразу, дата определяется из текста. Если дата неоднозначна, бот задаёт короткий уточняющий вопрос.

## Экспорт

`/export` готовит:

- CSV с плоской сводкой;
- XLSX с листами для основных разделов;
- JSON с полной историей.

Файлы должны удаляться по истечении `EXPORT_RETENTION_HOURS`.

## Политика медиа

Голосовые и фото сохраняются только на срок, необходимый для обработки. Срок задаётся `MEDIA_RETENTION_DAYS`. В обычные логи не пишутся тексты сообщений, медицинские детали и содержимое медиа.

## Деплой

Основной путь описан в [docs/deploy-render.md](docs/deploy-render.md). В проекте есть `render.yaml`: Render сам создаст web service и PostgreSQL, а секретные переменные попросит заполнить в интерфейсе.

## Ограничения MVP

- Нет интеграции с Flo, Apple Health, Garmin.
- Нет медицинских диагнозов и назначений.
- Калории не считаются точно без явных данных.
- Для production нужен реальный HTTPS endpoint и заполненные секреты.
